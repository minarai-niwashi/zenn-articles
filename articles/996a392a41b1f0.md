## 問題

VS Code で開いた `.ipynb` ファイル（ノートブック）を Claude Code に編集させると、次の2つの事象が起きることがあります。

- 編集が画面に反映されない
- 保存しようとすると次のようなダイアログが出る

> `'xxx.ipynb' を保存できませんでした: ファイルの内容がより新しいものです。ファイルを変更した内容で上書きしますか?` ［上書き／元に戻す］

## 原因と対策

どちらも、**ノートブックが未保存（タブに ●）のまま、Claude Code がディスク上のファイルを書き換えたとき**に起きます。

未保存の変更が残っていると、VS Code はそれを失わないよう、ディスクの新しい内容を画面に反映しません。さらに、その変更を保存しようとすると、ディスク側がすでに新しいため、上書きするか破棄するかを問うダイアログが出ます。

ノートブックでは、セルを実行するとその出力がセルに書き込まれ、**実行した時点で未保存（タブに ●）になります**。そのため「セルを実行 → 結果を見て Claude Code に直しを頼む」という自然な流れが、そのまま「未保存のままディスク上のファイルを書き換える」状況を作るため、事象が発生しやすくなります。

対策は、**Claude Code が書き換える前にノートブックを保存しておく**ことです。失われる未保存の変更がないので、VS Code はディスクの新しい内容を自動的に画面に反映します。手動で保存してもよいのですが、`settings.json` で `files.autoSave` を `onFocusChange` にすると保存を自動化できます。

```json
{
  "files.autoSave": "onFocusChange"
}
```

`onFocusChange` は、編集中のファイルからフォーカスが外れたときにそのファイルを保存する設定です。Claude Code に指示を打つのはノートブックからフォーカスを外す操作なので、その時にノートブックが保存されます。この設定で、私の環境では前述の2つの事象は起きなくなりました。
